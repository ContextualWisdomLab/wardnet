# Commercial Sale Readiness Plan

## Acceptance Criteria

1. Define the 2B KRW sale readiness threshold in committed documentation.
2. Implement license-aware commercial metadata APIs.
3. Implement threat feed import and feed status APIs.
4. Implement readiness and support bundle APIs.
5. Add Docker, Compose, and Kubernetes deployment assets.
6. Add security, compliance, product design, Figma, and buyer due-diligence docs.
7. Update smoke testing to prove license, feed import, readiness, support bundle, DNSBL, and restart persistence.
8. Preserve 100% line coverage for Rust code.
9. Publish the work through a GitHub PR and merge only after required checks are green or a structural blocker is proven.
10. Verify default branch CI/Scorecard and zero open PRs after merge.
